# Phase 2 — UI Design Contract: Homepage Rails

> Visual and interaction contract for the YouTube-style homepage (hero carousel + stacked category rails). Generated by gsd-ui-researcher, verified by gsd-ui-checker. Honors all LOCKED decisions in `02-CONTEXT.md`. Dark, cinematic, film-forward (isotopefilms / samhendi / yvonnerusso); CSS depth/tilt, no WebGL.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (SvelteKit 5 + Tailwind v4 — not a React/shadcn project; shadcn gate **not applicable**) |
| Preset | not applicable |
| Component library | none (hand-built Svelte components; adapt `_three`'s `VideoCard`, `PosterImage`, hero/rail mechanics) |
| Icon library | inline SVG only (chevrons drawn inline — no icon dependency added) |
| Font | Source Serif 4 (display, `--font-display`), Inter (body/UI, `--font-sans`), JetBrains Mono (`--font-mono`, badges only) — all self-hosted per Phase 1 D-13 |

**Token source of truth:** `src/app.css` `@theme` block. This spec references **real** token names — no new color tokens are introduced. Two new CSS custom properties for motion are specified in [§ Motion Tokens](#motion-tokens).

**Registry Safety:** No third-party registries. Vetting gate **not applicable**.

## Spacing Scale

8-point scale (multiples of 4). Map to Tailwind v4 spacing utilities (`p-*`, `gap-*`, where `4 = 1rem`).

| Token | Value | Tailwind | Usage on this page |
|-------|-------|----------|--------------------|
| xs | 4px | `1` | Dot gap, badge inset, title→meta gap |
| sm | 8px | `2` | Card gap inside a rail (mobile), chevron icon inset |
| md | 16px | `4` | Card gap inside a rail (desktop), rail title→track gap, page gutter (mobile) |
| lg | 24px | `6` | Page gutter (tablet+), hero overlay padding (mobile) |
| xl | 32px | `8` | Vertical gap between stacked rails, hero overlay padding (desktop) |
| 2xl | 48px | `12` | Hero bottom → first rail; section breaks |
| 3xl | 64px | `16` | Page top padding under nav (desktop), page bottom before footer |

**Exceptions:**
- Keyboard/touch target floor **44px** for chevron buttons and dot hit-areas (visual dot may be smaller; hit-area padded to 44px).
- Hero is **full-bleed** (edge-to-edge, no gutter); rails and rail headers are **contained** to `--page-gutter`.

`--page-gutter`: `16px` (mobile) → `24px` (≥640px) → clamp to keep content within `--content-max` on wide screens.

## Typography

Exactly **4 sizes / 2 families / 3 weights** in use on this page. Inter weights 400/500/600 and Source Serif 4 weights 400/600/700 are loaded (Phase 1); this page uses the subset below.

| Role | Family (token) | Size | Weight | Line Height | Used for |
|------|----------------|------|--------|-------------|----------|
| Display | `--font-display` (Source Serif 4) | `clamp(28px, 5vw, 48px)` | 700 | 1.1 | Hero slide title |
| Heading | `--font-display` (Source Serif 4) | 20px | 600 | 1.2 | Rail category labels |
| Body / Card title | `--font-sans` (Inter) | 16px | 500 | 1.4 | VideoCard title (clamped 2 lines) |
| Label / Meta | `--font-sans` (Inter) | 14px | 400 | 1.5 | Hero category eyebrow, card uploader/meta |
| Badge (mono) | `--font-mono` (JetBrains Mono) | 12px | 400 | 1 | Duration badge only |

- Body line-height **1.4–1.5**; heading/display **1.1–1.2** (cinematic, tight).
- Hero eyebrow (category name) is **uppercase**, `letter-spacing: 0.08em`, accent-colored via `categoryAccent(category)`.
- Card titles: `-webkit-line-clamp: 2`, `text-overflow: ellipsis`; never wrap past 2 lines.

## Color

Dark canvas. 60/30/10 split expressed in real tokens. **No new color tokens** — accent is the 8 per-category accents already defined.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--color-neutral-950` | `oklch(0.16 0 0)` | Page background, hero letterbox, gutters |
| Secondary (30%) | `--color-neutral-900` / `--color-neutral-800` | `oklch(0.22 0 0)` / `oklch(0.3 0 0)` | Card poster placeholder, rail track rest state, chevron button face, hero gradient scrim |
| Foreground | `--color-neutral-50` / `--color-neutral-300` | `oklch(0.98 0 0)` / `oklch(0.82 0 0)` | Titles (50), meta/uploader (300) |
| Accent (10%) | `--color-cat-*` (8) | per category | **Reserved list below** |
| Focus ring | `--ring-focus` / `--ring-focus-inner` | `oklch(0.98 0.02 80)` / `oklch(0.16 0 0)` | Global `:focus-visible` double ring (inherited, do not override) |

**Accent reserved for (explicit — never "all interactive elements"):**
1. Card hover/focus **ring** — `categoryAccentRing(category)` (`ring-cat-*/40`).
2. Hero slide **eyebrow** (category label text) — `categoryAccent(category)` (`text-cat-*`).
3. Rail label **left tick** — a 3px-wide accent bar before each rail title — `bg-cat-*` (full alpha) or `categoryAccentBg` for low-alpha fill.

Accent is **not** used for chevrons, dots, body text, or backgrounds at full alpha elsewhere. Chevrons and dots are neutral (`--color-neutral-50`/`-300`) so the page reads as one cinematic surface with category color as a quiet wayfinding cue.

**Contrast:** Card/hero titles use `--color-neutral-50` over poster imagery; hero title sits over the bottom scrim gradient (below) guaranteeing ≥ 4.5:1. Meta text `--color-neutral-300` over `-950` ≈ 7:1.

## Layout & Structure

```
+========================================================+  full-bleed
|                   HERO CAROUSEL                        |  edge-to-edge
|   poster (active, eager)  +  bottom scrim + title      |
|                    • • ● • • •   (dots)                |
+========================================================+
                       ↕ 48px (2xl)
+--------------------------------------------------------+  contained
| ▎ Category label (count desc order #1)        ‹  ›     |  max-width
|  [card][card][card][card][card]→ scroll-snap            |
+--------------------------------------------------------+
                       ↕ 32px (xl)
| ▎ Category label #2                            ‹  ›     |
|  [card][card][card]→                                    |
+--------------------------------------------------------+
                       ... one rail per category ...
                       ↕ 64px (3xl) → Footer
```

- **Page wrapper:** `<main>` on `--color-neutral-950`. Top padding `64px` desktop / `clamp` down on mobile (nav is `--chrome-nav-height` = 3.5rem, already sticky from Phase 1).
- **Content max-width:** `--content-max: 1440px`, centered; rails contained, hero full-bleed.
- **Rail stack gap:** `32px` (xl) between rails; `48px` (2xl) hero→first rail; `64px` (3xl) last rail→footer.
- **Section semantics:** hero is `<section aria-label="Featured work">`; the rail stack is `<section aria-label="Browse by category">` containing one `<section aria-labelledby="rail-{slug}">` per category, each with a heading element for its label.

### Breakpoints (Tailwind v4 defaults)

| Range | Cards visible per rail (approx) | Card width | Gutter | Hero height |
|-------|--------------------------------|------------|--------|-------------|
| Mobile `<640px` | ~1.3 (peek next) | `clamp(150px, 60vw, 220px)` | 16px | `56vh` min `360px` |
| Tablet `640–1024px` | ~3–4 | `200px` | 24px | `60vh` |
| Desktop `≥1024px` | ~5–6 | `220px` | 24px (clamped to max) | `clamp(440px, 64vh, 640px)` |

Card width is fixed-per-card (rail is a horizontal track, not a responsive grid); rail shows a partial next card on mobile to signal scrollability.

## Hero Carousel Spec

Source: `getHeroSlides()` (`$lib/data/heroSlides`) — slide 0 is the producer reel (`producerReelId`), then ≤6 representative non-reel slides (HERO_SLIDE_CAP = 6) in display order. **This variant is poster + title only — no autoplaying iframe** (LCP protection, CONTEXT lock).

| Property | Value |
|----------|-------|
| Container | full-bleed; height per breakpoint table above; `overflow: hidden`; `position: relative` |
| Poster | `getPosterFor(slide)`; `object-fit: cover`, `object-position: center`; active slide poster **`loading="eager"` + `fetchpriority="high"`**; all non-active posters `loading="lazy"` |
| Scrim | bottom-anchored gradient: `linear-gradient(to top, oklch(0.16 0 0 / 0.92) 0%, oklch(0.16 0 0 / 0.55) 28%, transparent 60%)` — guarantees title contrast |
| Overlay padding | `32px` desktop / `24px` mobile (lg/xl); content bottom-left aligned |
| Eyebrow | category name, uppercase, `letter-spacing 0.08em`, 14px Inter 400, `categoryAccent(category)` color |
| Title | `clamp(28px, 5vw, 48px)` Source Serif 4 **700**, `--color-neutral-50`, line-height 1.1, max 2 lines |
| Click target | entire active slide is a single `<a href="/watch/{id}">` wrapping the poster+overlay (label = slide title) |
| Dots | one per slide, bottom-center, gap `4px`, 44px hit-area; rest = 8px circle `--color-neutral-500`; active = 24px-wide pill `--color-neutral-50`; each dot is a `<button>` with `aria-label="Go to slide N: {title}"` and `aria-current="true"` on active |
| Auto-advance | every **7000ms**; resets on manual dot activation |
| Transition | crossfade `opacity` over **600ms** `cubic-bezier(0.4, 0, 0.2, 1)`; outgoing poster fades out as incoming fades in; **no slide/translate** (reduces layout/LCP cost). Optional incoming `scale(1.04 → 1)` Ken-Burns over 7s, motion-gated |
| Pause | auto-advance pauses on `pointerenter`/`focusin` within hero, resumes on `pointerleave`/`focusout` |
| Reduced motion | when `motion.prefersReducedMotion` → **no auto-advance, no crossfade/Ken-Burns**; render slide 0 statically; dots remain functional for manual nav; switching shows instant (no transition) |

**Focus order in hero:** slide link → dot 1 … dot N (left-to-right). Dots are reachable by Tab; Left/Right arrow keys on a focused dot move to prev/next dot and select that slide.

## Rail Spec

One rail per category from `getCategoriesInDisplayOrder()` (count desc, ties alpha). Cards from `getByCategory(category)`.

| Property | Value |
|----------|-------|
| Header | `<h2 id="rail-{slug}">` — 20px Source Serif 4 600, `--color-neutral-50`; preceded by a 3px-wide, full-height **accent tick** (`bg-cat-{slug}`); header→track gap `16px` |
| Track | horizontal flex; `overflow-x: auto`; `scroll-snap-type: x mandatory`; `scrollbar-hide` (existing `@utility`); `scroll-padding-inline: var(--page-gutter)` |
| Card gap | `8px` mobile / `16px` (md) tablet+ |
| Snap | each card `scroll-snap-align: start` |
| Touch | native horizontal drag/scroll (no JS needed on touch); momentum from platform |
| Chevrons | left/right `<button>`, desktop-only (`≥1024px`, `@media (hover: hover)`); 44×44px hit-area, 40px visible circle `--color-neutral-800` face at `0.8` alpha + `backdrop-blur`, chevron glyph inline SVG `--color-neutral-50`; vertically centered on the track; left at `inset-inline-start: 8px`, right at `inset-inline-end: 8px`; appear on rail hover/focus-within (`opacity 0→1`, 150ms) |
| Chevron action | scroll the track by ~one viewport-width of cards (`clientWidth * 0.8`) with `behavior: smooth` (or instant under reduced motion) |
| Arrows hide at extremes | left chevron hidden when `scrollLeft <= 0`; right chevron hidden when `scrollLeft + clientWidth >= scrollWidth - 1`; recompute on scroll + resize; chevrons get `aria-hidden` + `tabindex=-1` when hidden |
| Few-card rails | if total card width ≤ track width → **no overflow, no chevrons** (track left-aligned, not stretched) |
| Lazy loading | first rail's first ~4 posters may render normally; **all other posters `loading="lazy"`**; only the hero active poster is eager |

**Keyboard map (rail):**

| Key | Action |
|-----|--------|
| `Tab` | enters the rail at the first card link; moves card→card |
| `→` / `←` | (when a card is focused) move focus to next/prev card and `scrollIntoView({ inline: 'nearest' })` |
| `Home` / `End` | focus first / last card in the rail and scroll to it |
| `Enter` / `Space` | activate focused card → navigate to `/watch/{id}` (native `<a>` behavior; Space prevented-default then `.click()`) |

Focused card shows the **global double focus ring** (inherited `:focus-visible`) and is scrolled fully into view.

**Empty / edge states:**
- A category with zero non-hidden videos is **omitted entirely** (no empty rail rendered).
- If `getHeroSlides()` returns empty (data degenerate), hero collapses and rails render alone — no broken hero shell. (Acceptance criterion below.)

## VideoCard Spec

Real link, cinematic poster card with CSS-only depth. Adapt `_three`'s `VideoCard.svelte` + `categoryAccent` map.

| Property | Value |
|----------|-------|
| Element | `<a href="/watch/{video.id}">` — the whole card; `aria-label="Watch {title}"` |
| Aspect ratio | **16/9** poster (`aspect-[16/9]`) |
| Width | per breakpoint table (`clamp(150px,60vw,220px)` → `200px` → `220px`) |
| Radius | `12px` (`rounded-xl`) on poster + card |
| Poster | `getPosterFor(video)`; `object-fit: cover`; placeholder bg `--color-neutral-900` until load; `loading="lazy"` except as noted in Rail spec; `decoding="async"`; `alt="{title}"` |
| Duration badge | only if `video.duration_seconds` present; bottom-right, `4px` inset; 12px JetBrains Mono on `oklch(0.16 0 0 / 0.78)` pill, `--color-neutral-50` text, `4px` radius, formatted `m:ss` |
| Title | below poster, 16px Inter 500, `--color-neutral-50`, 2-line clamp, `8px` top gap |
| Meta (optional) | uploader/category, 14px Inter 400, `--color-neutral-300` |
| Perspective | parent rail track sets `perspective: 800px` so child tilt has depth |

### Hover / focus states — EXACT values

**Rest:** `transform: none;` `box-shadow: 0 1px 2px oklch(0.16 0 0 / 0.4);` no ring; title `--color-neutral-300`.

**Hover / focus-visible (full motion):**
- **Tilt:** `rotateX(var(--tilt-x)) rotateY(var(--tilt-y)) scale(1.03)` where `--tilt-x`/`--tilt-y` are set from cursor position, **max ±6deg** each (clamp). On keyboard focus (no cursor) → tilt = `0deg` but scale + ring + shadow still apply.
- **Lift shadow:** `box-shadow: 0 12px 28px -8px oklch(0.16 0 0 / 0.7), 0 2px 6px oklch(0.16 0 0 / 0.5);`
- **Accent ring:** apply `categoryAccentRing(category)` → `ring-cat-{slug}/40` at **2px** (`ring-2`), with `ring-offset-2 ring-offset-neutral-950`.
- **Brighten:** poster `filter: brightness(1.06);`
- **Title fade-up:** title (and meta) translate `4px → 0` and color `neutral-300 → neutral-50` over **180ms**; opacity `0.85 → 1`.
- **Transition:** `transform 180ms cubic-bezier(0.4,0,0.2,1), box-shadow 180ms, filter 180ms`. Tilt itself follows cursor with no transition (direct) for responsiveness; the scale/return uses the 180ms ease.

**Reduced motion variant** (`motion.prefersReducedMotion === true`):
- **No tilt, no scale, no Ken-Burns.** `transform` stays `none`.
- Hover/focus = `filter: brightness(1.08)` + `categoryAccentRing` (2px) + lift shadow only.
- Title color change still applies (instant, no translate).

**The single source of truth for the motion branch is `motion.prefersReducedMotion`** (the existing `$lib/state/motion.svelte.ts` rune). No component reads `matchMedia` directly.

## Motion Tokens

Two new CSS custom properties (set per-card from JS pointer handler; default 0):

| Token | Default | Range | Set by |
|-------|---------|-------|--------|
| `--tilt-x` | `0deg` | `-6deg … 6deg` | card `pointermove` handler (skipped when reduced-motion) |
| `--tilt-y` | `0deg` | `-6deg … 6deg` | card `pointermove` handler (skipped when reduced-motion) |

Shared easing: `--ease-cinematic: cubic-bezier(0.4, 0, 0.2, 1)` (add to `:root` in `app.css`). Durations: card 180ms, hero crossfade 600ms, chevron fade 150ms, auto-advance interval 7000ms.

Global reduced-motion backstop (in addition to the rune) — add to `app.css`:
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.001ms !important;
    transition-duration: 0.001ms !important;
    scroll-behavior: auto !important;
  }
}
```

## Accessibility

- **Semantics:** every card and hero slide is a real `<a href="/watch/{id}">`. Rail headings are real heading elements (`<h2>`); rail sections use `aria-labelledby`.
- **Focus ring:** inherit the **global `:focus-visible` double ring** (cream outline + dark inner box-shadow) from `app.css` — do **not** override per component. Card focus also scrolls the card fully into the rail viewport.
- **Keyboard:** full rail map above; dots arrow-navigable; nothing is mouse-only (chevrons are an enhancement; rails are fully keyboard-operable without them).
- **Reduced motion:** single rune (`motion.prefersReducedMotion`) gates hero auto-advance, hero crossfade/Ken-Burns, and card tilt/scale; plus the global CSS backstop. Reduced-motion users get a fully static, fully navigable page.
- **Contrast:** titles `--color-neutral-50` over scrim/`-950` ≥ 4.5:1; meta `--color-neutral-300` over `-950` ≈ 7:1; accent rings are decorative (state also conveyed by shadow/brightness, not color alone).
- **Alt text:** poster `alt = video.title`; decorative gradient/scrim has no alt; dots/chevrons have `aria-label`s above.
- **Target size:** chevrons and dots ≥ 44×44px hit-area.
- **LCP:** only the active hero poster is `loading="eager" fetchpriority="high"`; everything else `loading="lazy"`. No autoplaying iframe in the hero for this variant.

## Acceptance Criteria (gsd-ui-checker — grep-able where possible)

**Tokens / no-magic-color:**
1. No raw hex or `rgb(` colors in homepage components — all colors via `--color-*`, `--ring-*`, or `text-cat-*/bg-cat-*/ring-cat-*` utilities. (`grep -rE "#[0-9a-fA-F]{3,6}|rgb\(" src/routes/+page.svelte src/lib/components/VideoCard.svelte` → no matches.)
2. Card accent ring uses the map, not literals: `categoryAccentRing(` appears in `VideoCard.svelte`. New `--ease-cinematic` and the reduced-motion backstop exist in `src/app.css`.

**Hero:**
3. `getHeroSlides` imported in the homepage; active poster has `loading="eager"` and `fetchpriority="high"`; non-active posters `loading="lazy"`. (`grep` for `fetchpriority="high"` → exactly one in hero.)
4. Auto-advance interval is `7000`; crossfade not a translate. Dots are `<button>` with `aria-label` and `aria-current`.
5. Hero references `motion.prefersReducedMotion` (no direct `matchMedia` in `+page.svelte`/hero component).

**Rails:**
6. Rail order is `getCategoriesInDisplayOrder()`; cards from `getByCategory()`. Track uses `scroll-snap-type: x mandatory`, `scroll-snap-align`, and the `scrollbar-hide` utility.
7. Chevrons are `<button>` with `aria-label`, ≥44px hit-area, hidden (`aria-hidden`+`tabindex=-1`) at extremes; rails that don't overflow render no chevrons.
8. Empty categories omitted (no rail with 0 cards in DOM).

**Card:**
9. Card root is `<a href` to `/watch/{id}` with `aria-label="Watch …"`; poster `aspect-[16/9]`, `rounded-xl`, `alt={title}`, `loading="lazy"` (except hero-region eager).
10. Tilt clamped to ±6deg via `--tilt-x`/`--tilt-y`; reduced-motion branch sets transform `none` (verifiable in component logic + a unit/behavior test toggling `__setPrefersReducedMotionForTests`).

**Accessibility / motion:**
11. No per-component `outline`/`box-shadow` override of `:focus-visible` (global ring inherited).
12. With `prefers-reduced-motion: reduce`: no `setInterval` advance fires, no tilt transform applied, dots still navigate. (Behavioral test using the existing motion-rune test hooks.)

**Build / perf:**
13. `pnpm build` stays green; prerender succeeds; `/watch/[id]` links resolve (either minimal stub route added this phase or forward-route allow-list retained — planner flags which).
14. No WebGL / Three.js / `<canvas>` introduced (CSS transforms only).
